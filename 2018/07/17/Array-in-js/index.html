JS数组方法

1.Array.from():从一个类似数组或可迭代的对象中创建一个数组实例：

Demo

语法：Array.from(arrayLike[,mapFn[,thisArg]]);arrayLike:想要转换成数组的伪数组对象或可迭代对象；mapFn(可选参数):如果指定了该参数，新数组中的每个元素会执行该回调函数；thisArg(可选参数):可选，执行回调函数mapFn时this对象。该方法返回一个新的数组实例。

有一点就是 Array.from.length = 1.

Demo

好的出现了Set这个方法了，嗯，我不懂。来探讨一下：

一个简单的demo

是不是很简单？可是这是很浅的一部分呢！Set对象允许你存储任何类型的唯一值，无论是原始值或者是对象引用。，它语法看上去好像很简单：new Set([iterable]),其中iterable参数:如果传递一个可迭代对象，它的所有元素都将被添加到新的Set中。如果不指定此参数或其值为null，则新的Set为空，这个方法返回的是一个Set对象。

属性:Set.prototype.size：返回Set对象的值的个数；方法有很多(详情见mdn);

from()的demo还有用到Map的：

语法：let new_array = arr.map(function callback(currentValue,index,array){
// Return element for new_array
}[,thisArg])
其中，callback：生成新数组元素的函数，使用三个参数；
currentValue：callback的第一个参数，数组中正在处理的当前元素。index：callback的第二个参数，数组中正在处理的当前元素的索引。array：callback的第三个参数，map方法被调用的数组。 thisArg，可选的，执行callback函数时使用的this值。 返回值！！！这个务必要注意一下，因为map()方法跟forEach()方法差不多，但map有返回值，forEach没有。 

通常情况下，map 方法中的 callback 函数只需要接受一个参数，就是正在被遍历的数组元素本身。但这并不意味着 map 只给 callback 传了一个参数。这个思维惯性可能会让我们犯一个很容易犯的错误。
